## Core Principles

### 1. Tests as Agent Constraints
Tests aren't just for code qualityâ€”they're **executable specifications** that constrain agent behavior. Without tests, agents drift. With tests, they have clear success/failure signals.

### 2. Defense in Depth
Don't rely on a single enforcement mechanism:
- **CLAUDE.md**: Soft guidance (can be ignored)
- **Hooks**: Runtime enforcement (deterministic)
- **CI/CD**: Pipeline gates (final checkpoint)

### 3. Self-Improvement Loops
Every mistake should make the system smarter:
```
Agent Error â†’ Session Log â†’ Analysis â†’ CLAUDE.md Update â†’ Better Agent
```

### 4. Modular Configuration
Use `@path/to/file.md` imports in CLAUDE.md to keep context focused and token-efficient.

### 5. Accuracy Over Comprehensiveness
In a fast-moving ecosystem, outdated information causes more harm than gaps. Verify claims against recent sources; acknowledge uncertainty.

## Document Naming Convention

All documents follow this pattern:
```
{topic}-guide-{updated-}{YYYY-MM-DD}.md
```

- **topic**: Kebab-case topic name
- **updated**: Optional prefix indicating revision of earlier version
- **date**: Last significant update date

When updating a document, change the date suffix to the current date.

## Contributing / Updating

Each document is self-contained. To update:
1. Edit the specific document
2. Update the `Last Updated` date in the document header
3. Update filename date suffix if making significant changes
4. Update this index if filenames change
5. Follow methodology in [documentation-methodology-guide](./documentation-methodology-guide-2025-12-10.md)
